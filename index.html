SPM12学习2

概念

  1. TR: 重复时间即，脉冲序列执行所需的时间
  2. TA: 采集时间也称扫描时间，即整个脉冲序列完成信号采集所需要的时间
  3. 图像配准：Image registration，将不同时间、不同传感器或不同条件下获取的两幅图或者多幅图像进行匹配、叠加的过程
  4. QSM: 磁共振采集得到的磁敏感定量图像（Quantitative Susceptibility Mapping）
  5. 傅里叶变换：任何连续周期信号可以由一组适当的正弦曲线组合而成
  6. hippocampal atrophy：海马体萎缩
  7. functional connectivity:简称FC
  8. MMSE??????
  9. independent component analysis:简称ICA即独立成分分析法，用于发现存在于随机变量下的隐性因素
  10. clustering coefficient：集聚系数
  11. dual regression: 二元回归
  12. AUC：全称area under the curve，是机器学习的评估指标用来评估二分类模型的性能指标
  13. ALFF：全称amplitude of low frequency fluctuations即低频震荡振幅算法
  14. default mode network:静息状态默认脑活动网络
  15. time course VS time series:查询链接“time course experiment data series.”太长，所以简写为”time series data”
  16. sliding windows approach: “a-brute-force search over pose space with fixed model to find object”暴力搜索
  17. confidence map:置信图
  18. k-means clustering：k均值聚类
  19. Manhattan Distance:曼哈顿距离，两点在南北方向上的距离加上在东西方向上的距离，即d（i，j）=|xi-xj|+|yi-yj|
  20. FSL：FMRIB Software Library(FSL) 以下来自https://fsl.fmrib.ox.ac.uk/fsl/fslwiki/官网介绍，FSL是一个用于分析fMRI、MRI、DTI(弥散张量成像)的综合库工具。
  21. cross validation:交叉验证，用于评估模型。可以用来解决过拟合。如何解决呢？在训练集的基础上划分出新的训练集与验证集，新训练集训练模型，在验证集测试模型，不断调整初始模型（超参数等），使得训练得到的模型在验证集上的表现最好，最后放到测试集上得到这个最优模型的评估结果。模型的表现过分依赖于验证集的划分，可能使某些特殊样本被划入验证集，导致模型的表现出现异常（偏好或偏差）。而且训练集划了一部分给验证集后，训练模型能得到的数据就变少了，也会影响训练效果。因为通常来说，训练数据越多，越能反映出数据的真实分布，模型训练的效果就越好，越可能得到无偏估计。
  22. nested cross validation:嵌套交叉验证，这样可以使模型在全部数据上进行测试，能更好的说明模型的泛化能力
  23. T-statistic,Z-statistic:T统计量，Z统计量

BOLD-fMRI数据分析基础

界面中的参数

  • Realign:Estimate
  1. 通过使用最小二乘法和6个刚体变换对同一对象的时间序列图像就行重新整合
  2. 通过user指定序列中的具体图像作为第一张图像，之后的所有扫描子序列能够被重新组合
  3. 质量与重新组合的速度的权衡，Separation此参数以毫米为单位，表示对参考图像文件进行重采样时采样点之间的间隔，采样距离越小，得到的结果越精确，但是运行速度越慢
  4. FWHM称为脉冲的半高宽度，高斯平滑的FWHM被用于评估图像的重排参数之前

  • 若要按值对 Series 进行排序，当使用 .order(na_last=True, ascending=True, kind=’mergesort’) 方法，任何缺失值默认都会被放到 Series 的末尾。

冒泡排序 $$O(n^2)$$ 0 360 1437 8927 36719

快速排序 $$O(nlogn)$$ 0 0 16 32 63

算法评价

  • 虽然冒泡排序和选择排序均是$$O(n^2)$$但是每个的性能还是有很大的差异，冒泡排序采用每次比较然后再改变顺序，而选择排序则采用， 记录当前待排序的最大值或者最小值，执行完第n次大循环在将最大值或者最小值插入到指定的序列中
  • 快速排序的基本思想采用分而治之，将分为左右两段进行递归提高了效率

Git

Git基本语法

  • $ git init 将当前的目录变成Git可以管理的仓库
  • $ ls -ah显示当前目录下的所有文件包括隐藏文件
  • $ git add readme.txt 将readme.txt 添加到仓库
  • $ git commit -m "wrote a readme file" 将文件提交到仓库 并添加本次提交的说明

版本回退

*$ git status 查看仓库当前的状态

  • $ git diff readme.txt查看之前修改的内容
  • $ git log 版本控制的历史记录
    用
  • 用 HEAD表示当前版本, HEAD^表示上个版本,100个版本之前HEAD~100 回退到上一个版本为
    $ git reset --hard HEAD^
  • 回到之前的那个版本也就是未被修改的 找到之前bash 窗口中的git log 第一个版本的信息的乱七八糟的数字字母 $ git reset --hard 你的版本码
  • 如果关掉电脑重启怎么办?
    $ git reflog 查看历史命令

说明

  • git管理的是修改并非文件，因此每次的修改都必须先add再commit,add到暂存区的才能被git提交
  • 撤销修改
  1. 在未add前使用git check -- file让file回到最近的一次git commit 或者git add时的状态(丢弃工作区的修改)
  2. add之后使用git reset HEAD file将暂存区的修改回退到工作区再使用git checkout -- file回退 –
