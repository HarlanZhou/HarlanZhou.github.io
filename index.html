各算法运行时间比较

算法 时间复杂度 100数排序 1000数排序 2000数排序 5000数排序 1W数排序
冒泡排序 $$O(n^2)$$ 0 360 1437 8927 36719
选择排序 $$O(n^2)$$ 0 171 625 3953 17281
快速排序 $$O(nlogn)$$ 0 0 16 32 63

算法评价

  • 虽然冒泡排序和选择排序均是$$O(n^2)$$但是每个的性能还是有很大的差异，冒泡排序采用每次比较然后再改变顺序，而选择排序则采用， 记录当前待排序的最大值或者最小值，执行完第n次大循环在将最大值或者最小值插入到指定的序列中
  • 快速排序的基本思想采用分而治之，将分为左右两段进行递归提高了效率

Git

Git基本语法

  • $ git init 将当前的目录变成Git可以管理的仓库
  • $ ls -ah显示当前目录下的所有文件包括隐藏文件
  • $ git add readme.txt 将readme.txt 添加到仓库
  • $ git commit -m "wrote a readme file" 将文件提交到仓库 并添加本次提交的说明

版本回退

*$ git status 查看仓库当前的状态

  • $ git diff readme.txt查看之前修改的内容
  • $ git log 版本控制的历史记录
    用
  • 用 HEAD表示当前版本, HEAD^表示上个版本,100个版本之前HEAD~100 回退到上一个版本为
    $ git reset --hard HEAD^
  • 回到之前的那个版本也就是未被修改的 找到之前bash 窗口中的git log 第一个版本的信息的乱七八糟的数字字母 $ git reset --hard 你的版本码
  • 如果关掉电脑重启怎么办?
    $ git reflog 查看历史命令

说明

  • git管理的是修改并非文件，因此每次的修改都必须先add再commit,add到暂存区的才能被git提交
  • 撤销修改
  1. 在未add前使用git check -- file让file回到最近的一次git commit 或者git add时的状态(丢弃工作区的修改)
  2. add之后使用git reset HEAD file将暂存区的修改回退到工作区再使用git checkout -- file回退 –
